GET STARTED LOCALLY IN 5 STEPS
================================================================================

STEP 1: CLONE THE REPO
$ cd your-projects-folder
$ git clone https://github.com/withnaturehack/heatwave.git
$ cd heatwave
$ git checkout profile-management-fix

STEP 2: INSTALL DEPENDENCIES
$ npm install

STEP 3: GET YOUR API KEYS

You need 5 keys. Get them from:

A) SUPABASE KEYS
   1. Go to https://supabase.com
   2. Sign in to your project
   3. Click Settings → API
   4. Copy: Project URL and anon key

B) GOOGLE OAUTH KEYS
   1. Go to https://console.cloud.google.com
   2. Create OAuth credentials for your app
   3. Get: Client ID and Client Secret

C) ELEVENLABS API KEY
   1. Go to https://elevenlabs.io
   2. Go to Settings → API Keys
   3. Copy your API key

STEP 4: CREATE .env.local FILE

Create a file called `.env.local` in the project root:

NEXT_PUBLIC_SUPABASE_URL=https://your-project-url.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key_here
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret
ELEVENLABS_API_KEY=your_elevenlabs_api_key

Replace the values with your actual keys from Step 3.

STEP 5: SETUP DATABASE MIGRATIONS

5a) Open Supabase SQL Editor
    - Go to https://supabase.com
    - Click your project
    - Click SQL Editor (left sidebar)

5b) Run migrations in order (01, 02, 03, 04, 05, 06, 07)

For each migration file:
    1. Open: supabase/01-normalize-profiles-schema.sql
    2. Copy all the SQL code
    3. Paste into SQL Editor
    4. Click "Run"
    5. Wait for success
    6. Repeat for files 02 through 07

The migrations are idempotent (safe to re-run).

STEP 6: RUN THE APP

$ npm run dev

The app will start at: http://localhost:3000

Test it:
  1. Click "Sign in with Google"
  2. Complete onboarding
  3. Upload photos
  4. Check Supabase Dashboard to see data in tables

================================================================================
EXPECTED DATABASE BEHAVIOR
================================================================================

After signing up:

1. profiles table gets a new row with:
   - user_id (your auth ID)
   - is_complete: false
   - All other fields empty

2. membership table gets a new row with:
   - user_id (your auth ID)
   - status: pending
   - applied_at timestamp

3. As you complete onboarding:
   - display_name, social_link, photos saved
   - onboarding_answers updated with JSON data
   - is_complete set to true

4. Once approved (by admin):
   - membership.status changes to approved
   - members table gets a new row with unique referral code

5. When matched:
   - matches table gets a new row with match data
   - compatibility scores, strengths, tensions, dealbreakers

6. When messaging:
   - messages table gets rows with message content
   - read_at tracks when read

================================================================================
TROUBLESHOOTING
================================================================================

Problem: App won't start
→ Check npm install worked (try: npm install again)
→ Check .env.local has all 5 keys

Problem: Can't sign in with Google
→ Check Google Client ID is correct in .env.local
→ Check Google OAuth redirect URIs include localhost:3000

Problem: Database tables don't exist
→ Check all 7 migrations ran in Supabase SQL Editor
→ Look for error messages in SQL Editor

Problem: Photos won't upload
→ Check file is < 5MB
→ Check format is JPEG, PNG, WebP, or GIF

Problem: Voice doesn't work
→ Check ELEVENLABS_API_KEY in .env.local
→ Check key is valid in ElevenLabs dashboard
